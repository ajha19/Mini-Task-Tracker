2. Setup & Installation ‚öôÔ∏è

Follow these steps to run the application locally.

Prerequisites

  • Node.js (v18+)
  • MongoDB (Port 27017)
  • Redis (Port 6379)

Step 1: Backend Setup

cd backend
npm install
cp .env.example .env
# Ensure PORT=5001 is set in .env
npm run dev

Step 2: Frontend Setup

cd frontend
npm install
cp .env.example .env.local
npm run dev

The application will be accessible at http://localhost:3000.

3. Key Features üì±

Authentication & Security

  • Secure Login/Signup: JWT-based authentication with Bcrypt password hashing.
  • Auto-Logout: Users are automatically logged out if they navigate back to the login page to ensure session security.
  • Protected Routes: Dashboard is inaccessible without a valid token.

Task Management

  • CRUD Operations: Create, Read, Update, and Delete tasks seamlessly.
  • Due Dates: Assign dates to tasks to track deadlines.
  • Filtering: Filter tasks by All, Pending, or Completed status.
  • Optimistic UI: Instant visual feedback when toggling task status.

4. System Quality & Architecture üèóÔ∏è

The codebase is structured for scalability and maintainability:

  • Modular Architecture: Separation of Controllers, Services, and Routes.
  • Type Safety: 100% TypeScript implementation across frontend and backend.
  • Performance: Redis caching strategy implemented for the GET /tasks endpoint to reduce database load.
